版本：1.0

作者：胡智

---

## 1. 产品概述

### 1.1 定位
帮助用户从"死记硬背全文"转变为"理解核心、自主复述、自然脱稿"的轻量化口播训练工具。

### 1.2 核心价值
- AI智能拆解文稿 → 核心主旨 + 关键要点
- 4种阶梯式练习模式，逐步脱稿
- 录音回放，自我对比
- 社恐友好：无评分、无社交、无压力

### 1.3 目标用户
- 社恐/内向，需要练习表达的人
- 只能背稿、无法脱稿的人
- 需要结构化练习材料的人

---

## 2. 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 框架 | React 18 + Vite | 快速开发，HMR友好 |
| 样式 | Tailwind CSS | 极简治愈风实现简单 |
| 路由 | React Router v6 | SPA页面切换 |
| AI | 通义千问 API | 国内直连，配置文件管理 |
| 存储 | localStorage | 无需后端，历史记录本地存储 |
| 录音 | MediaRecorder API | 浏览器原生，无需依赖 |

---

## 3. 页面结构

### 3.1 路由设计

```
/                → 首页（输入文稿 + 历史快捷入口）
/practice/:id    → 练习页（4种模式Tab + 底部录音栏）
/history         → 历史记录列表页
/settings        → 设置页（API Key配置）
```

### 3.2 页面流程

```
首页输入 → AI拆解 → 练习页（4模式切换+录音）→ 返回首页/历史
                    ↓
              历史页（查看/继续/删除）
```

## 4. 页面设计

### 4.1 首页

**布局：** 居中卡片式

```
┌─────────────────────────────────────┐
│  [设置图标]                          │
│                                      │
│         口播脱稿练习                  │
│     帮你抓住重点，自然表达            │
│                                      │
│  ┌─────────────────────────────────┐ │
│  │                                 │ │
│  │     文本输入框                   │ │
│  │     (50-1000字)                 │ │
│  │                                 │ │
│  └─────────────────────────────────┘ │
│                                      │
│        [ 生成提纲 & 开始练习 ]        │
│                                      │
│  ─────── 最近练习 ───────            │
│  ┌─────────┐  ┌─────────┐           │
│  │ 历史1   │  │ 历史2   │           │
│  └─────────┘  └─────────┘           │
└─────────────────────────────────────┘
```

**交互：**
- 空内容提交 → 轻提示"请输入练习文稿"
- 提交后 → 加载动画 → 调用AI → 跳转练习页
- 点击历史卡片 → 直接进入练习页

### 4.2 练习页

**布局：**

```
┌─────────────────────────────────────┐
│  [←返回]                            │
│                                      │
│  [原文对照] [仅提纲] [半提示] [完全脱稿] │
│  ─────────────────────────────────── │
│                                      │
│         内容展示区                    │
│         (根据Tab切换)                │
│                                      │
│                                      │
│                                      │
├─────────────────────────────────────┤
│         [● 录音]  00:00              │  ← 底部固定录音栏
└─────────────────────────────────────┘
```

**4种模式内容：**

| 模式 | 显示内容 |
|------|----------|
| 原文对照 | 左侧原文 + 右侧AI提纲（双栏布局） |
| 仅提纲 | 核心主旨（大字）+ 关键要点列表 |
| 半提示 | 每段仅显示关键词/首词，其余隐藏 |
| 完全脱稿 | 空白 + 柔和提示"尝试脱稿复述" |

**录音栏功能：**
- 圆形录音按钮（点击开始/停止）
- 录音时长实时显示
- 录音后显示播放/重录按钮

## 5. AI接口设计

### 5.1 配置文件

```javascript
// src/config/ai.config.js
export const AI_CONFIG = {
  provider: 'qwen',  // 通义千问
  apiKey: '',        // 从localStorage读取或设置页配置
  endpoint: 'https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation',
  model: 'qwen-turbo'
};
```

### 5.2 Prompt模板

```
你是一个口播训练助手。请将用户输入的文稿拆解为以下结构：

1. 核心主旨（1句话，不超过30字）
2. 关键要点（3-6条，每条不超过20字，按原文顺序）
3. 口播复述提示（1-2句简单提示）

规则：
- 不润色、不扩写、不文艺化
- 不增加原文没有的信息
- 要点短句化、口语化
- 严格保留原文顺序

请以JSON格式输出：
{
  "mainIdea": "核心主旨",
  "keyPoints": ["要点1", "要点2", "要点3"],
  "tips": "复述提示"
}
```

### 5.3 数据结构

```typescript
interface PracticeItem {
  id: string;
  originalText: string;      // 原始文稿
  mainIdea: string;          // 核心主旨
  keyPoints: string[];       // 关键要点
  tips: string;              // 复述提示
  createdAt: number;         // 创建时间戳
  recordings?: Blob[];       // 录音数据（可选）
}
```

## 8. 非功能需求

### 8.1 性能
- AI拆解响应 ≤ 5秒
- 页面加载 ≤ 2秒
- 录音流畅不卡顿

### 8.2 兼容性
- PC端优先（Chrome、Edge、Firefox）
- 移动端自适应（后续迭代）

### 8.3 隐私
- 文稿与录音仅存本地
- 不收集用户信息
- API Key 本地存储
